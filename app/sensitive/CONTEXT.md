# 敏感信息监测

监测互联网上与目标单位相关、本不应公开的敏感数据泄露，经 AI 研判后供人工复核。与「仿冒网站发现」隔离，是独立功能（独立领域代码，仅通过 `app/common` 共享横切基础设施）。

## Language

**敏感信息监测**:
通过关键字在互联网上发现与目标单位相关的数据泄露线索、并由 AI 研判其真实性与敏感程度的活动。
_Avoid_: 舆情监测, 内容审核, 暗网监控（暗网只是其中一种来源，不等于整个活动）

**数据泄露**:
本应非公开的、与目标单位相关的资产被公开暴露或挂网售卖。例如员工账号口令、内部文档、源代码、数据库、API 密钥、敏感配置。
_Avoid_: 敏感信息（过于宽泛）, 负面舆情

**监测任务**:
一次敏感信息监测的执行单元，输入一组目标关键字，产出一批泄露线索。独立于仿冒站发现里的「任务」（独立的表与实体）。
_Avoid_: 任务（单说会与仿冒站的任务混淆）, Task

**目标关键字**:
监测任务的输入锚点，分四类：单位名/品牌词、域名/子域名、邮箱后缀、内部特征串。AI 研判「相关性」时以此为锚。
_Avoid_: 关键词（与仿冒站发现的「品牌词」混淆）, query

**泄露线索**:
来自某一来源、疑似与目标单位相关的一条数据泄露记录，含来源、定位、证据片段与 AI 研判结果。逐条独立研判，不跨源归并。
_Avoid_: 候选（那是仿冒站发现里的概念）, 命中, 结果

**定位**:
一条泄露线索在其来源中的唯一位置，用作去重键。网页源=URL，代码源=仓库+文件路径，网盘源=分享链接，情报源=记录 ID。
_Avoid_: URL（只覆盖网页源，不够通用）, 链接

**来源类型**:
泄露线索的产出渠道，MVP 枚举为：通用网页搜索、代码托管搜索、网盘文库搜索、暗网情报。可插拔，未配置则跳过。
_Avoid_: 数据源（与仿冒站发现的「发现源」混淆）

**研判结论**:
AI 对一条泄露线索的定性：确认泄露 / 疑似 / 无关（误报）。
_Avoid_: verdict 直译, 判定

**敏感等级**:
泄露线索的优先级：高 / 中 / 低。列表默认按此排序，取代仿冒站发现里的加权评分。
_Avoid_: 评分, score, 危害分

**泄露类型**:
AI 给出的内容分类：账号口令 / 源代码 / 内部文档 / 数据库 / 密钥 / 其他。
_Avoid_: 分类, 标签
